# 总体判断

**DeepSeek 在本批样本中整体更可靠。** 核心分歧集中在 Reason 维度的评判标准上：Kimi 大量将"知识解释"类问题的 Reason 维度判为 -1（不涉及），而 DeepSeek 认为即便是直接陈述型回答，只要逻辑连贯、方向与 evidence 一致即可给高分。DeepSeek 的处理更合理——Reason 维度评估的是回答中推理/逻辑是否连贯无矛盾，即便回答以直接陈述形式呈现，其内在逻辑仍可被评估，不应一律判为"不涉及"。

在 Completeness 和 Traceability 维度上，两者差异较小（通常 0.5-1.0 分），DeepSeek 倾向给更高分，Kimi 更保守但理由也合理。

# 结论

1. **Kimi 存在系统性问题**：对 Reason 维度的 -1 判定过于机械，将"无显式推理过程"等同于"不涉及"，导致大量有效评分丢失。这是本批分歧的主要来源（占比约 60%）。

2. **DeepSeek 在宽松标准下更一致**：对"超出 evidence 但方向一致"的内容处理更统一，给分逻辑自洽。

3. **Kimi 在 Traceability 维度的审慎有价值**：其对标注精度的关注比 DeepSeek 更细致，适合作为质量上限参考。

4. **建议动作**：优先对 Reason 维度的 -1 判定标准达成共识，这将一次性解决约 25 条分歧。
