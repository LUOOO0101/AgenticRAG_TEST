# 两轮分类差异分析报告

## 分类标准回顾

**RAG（直接检索）**：
- 问单一事实、定义、数值或描述
- 可用一句话或一个数字回答
- 只涉及单一维度的询问或对比

**Agentic RAG（多步推理）**：
- query中用顿号、逗号或"等"明确列举了多个不同方面
- query中包含多个独立问句
- 要求解释原因、机制、策略等需要深度分析
- 问"核心优势""关键因素""主要原因"等需要列举多个独立方面

## 统计总结

### 分类不一致的17条记录：

**应为 agentic rag 但第一轮或第二轮误判为 rag 的：12条**
- 原因：包含"总结""分别""如何""为何"等关键词，或列举多个方面，需要多步推理或多维度归纳

**应为 rag 但第一轮或第二轮误判为 agentic rag 的：5条**
- 原因：虽然涉及对比或列举，但属于单一维度的简单对比，一次检索即可完成

### 核心判断要点：

1. **"总结/梳理/概括"类问题**：如果要求列举多个独立方面 → agentic rag
2. **"A、B、C分别是什么"**：明确列举多个查询对象 → agentic rag
3. **"如何/怎么做"类问题**：需要描述多步骤流程 → agentic rag
4. **"为何/为什么"类问题**：需要解释原因和深度推理 → agentic rag
5. **单一维度对比**（如"在X方面有何不同"）→ rag
6. **固定答案的列举**（如"包括哪三部分"）→ 需要看是否需要从多处提取归纳，如果是固定结构则为 rag

---

## 建议

第一轮分类模型过于保守，将很多需要多步归纳的"总结"类问题误判为 rag；第二轮有所改进但在单一维度对比问题上出现过度分类。建议：

1. 强化对"总结/梳理/概括"关键词的识别
2. 对"分别"+"顿号列举"的模式给予更高权重
3. 单一维度对比（明确指定对比维度）应优先判定为 rag
